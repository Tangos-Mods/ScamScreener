# Minecraft 26.1 Port Check

This checklist is for the later ScamScreener port to `26.1`.

Status of this note: `2026-03-26`.

## Current Blocker

As of `2026-03-26`, `26.1` already has:

- Fabric Loader
- Fabric API
- ModMenu builds

But it still does not have the full official mapping state required for this repo port:

- no official `Yarn` for final `26.1`
- no clean usable `net.fabricmc:intermediary:26.1`

As long as that is still true, do not enable `26.1` in `settings.gradle.kts`.

## Go / No-Go

Only start the port once all of these are true:

- `Fabric Loader` exists for `26.1`
- `Fabric API` exists for `26.1`
- `Yarn` exists for final `26.1`
- `intermediary-26.1` is reachable on Fabric Maven
- optional `ModMenu` for `26.1` exists, or it is intentionally left optional/unset

If `Yarn` or `intermediary` are still missing: stop and check again later.

## Quick Check

All commands below are for PowerShell.

### 1. Confirm the Minecraft version

```powershell
Invoke-RestMethod https://meta.fabricmc.net/v2/versions/game |
  Where-Object { $_.version -eq '26.1' } |
  ConvertTo-Json -Depth 4
```

Expected:

- an entry for `26.1`
- `stable = true`

### 2. Check Fabric Loader

```powershell
Invoke-RestMethod https://meta.fabricmc.net/v2/versions/loader/26.1 |
  Select-Object -First 5 |
  ConvertTo-Json -Depth 4
```

Expected:

- at least one usable loader entry

### 3. Check Yarn for final 26.1

```powershell
Invoke-RestMethod https://meta.fabricmc.net/v2/versions/yarn/26.1 |
  ConvertTo-Json -Depth 4
```

Expected:

- the list is not empty

If the list is empty: postpone the port.

### 4. Check the intermediary artifact

```powershell
try {
  (Invoke-WebRequest -Method Head https://maven.fabricmc.net/net/fabricmc/intermediary/26.1/intermediary-26.1-v2.jar).StatusCode
} catch {
  $_.Exception.Response.StatusCode.value__
}
```

Expected:

- `200`

If this returns `404` or the metadata is still broken: postpone the port.

### 5. Check Fabric API

```powershell
$versions = Invoke-RestMethod 'https://api.modrinth.com/v2/project/fabric-api/version'
$versions |
  Where-Object { $_.game_versions -contains '26.1' -and $_.loaders -contains 'fabric' } |
  Select-Object -First 5 version_number, game_versions, date_published |
  ConvertTo-Json -Depth 4
```

Expected:

- at least one `fabric-api` version for `26.1`

### 6. Check ModMenu

```powershell
$versions = Invoke-RestMethod 'https://api.modrinth.com/v2/project/modmenu/version'
$versions |
  Where-Object { $_.game_versions -contains '26.1' -and $_.loaders -contains 'fabric' } |
  Select-Object -First 5 version_number, game_versions, date_published |
  ConvertTo-Json -Depth 4
```

Expected:

- at least one `modmenu` version for `26.1`

## Verification After the Port

At minimum, run:

```powershell
./gradlew.bat projects
./gradlew.bat :26.1:compileJava --no-daemon --stacktrace
./gradlew.bat :26.1:build --no-daemon --stacktrace
```

If the full multi-target build must stay green, then also run:

```powershell
./gradlew.bat build --no-daemon --stacktrace
```

- `:26.1:compileJava` is green
- `:26.1:build` is green
